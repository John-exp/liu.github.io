由于github无法动态加载内容（php），因此需要本地更新后上传，操作流程如下：

## 步骤一：准备文件

1. 准备一个文件夹：
   - 选择一个文件夹用来放置所需要的文件，例如 `图片展示`。

2. 添加图片和子文件夹：
   - 将您想展示的图片文件夹放入其中。例如：
     ```
     图片展示/
     │
     ├── 1/
     │   └── 1.PNG
     ├── 2/
     │   └── 2.PNG
     ├── 3/
     │   └── 3.PNG
     ├── 4/
     │   └── 4.PNG
     ├── 5/
     │   └── 5.PNG
     ```

3. 添加项目文件：
   - 将 `index.html` 和 `update-imgs.ps1` 文件复制到 `图片展示` 文件夹中，使其与图片子文件夹处于同一目录下。

	应与下方保持结构一致：

     ```
     图片展示/
     │
     ├── 1/
     │   └── 1.PNG
     ├── 2/
     │   └── 2.PNG
     ├── 3/
     │   └── 3.PNG
     ├── 4/
     │   └── 4.PNG
     ├── 5/
     │   └── 5.PNG
     ├── index.html
     └── update-imgs.ps1
     ```

## 步骤二：运行 PowerShell 脚本

1. 运行脚本：
   - 在文件夹中，找到 `update-imgs.ps1` 文件。
   - 右键点击 `update-imgs.ps1`，选择“使用 PowerShell 运行”。

2. 完成运行：
   - 脚本运行后，您应该会在 `图片展示` 文件夹中看到一个新的文件 `images.json`。

## 步骤三：上传文件到 GitHub

3. 上传文件：
   - 进入您刚刚创建的仓库页面。
   - 点击“Add file”按钮，选择“Upload files”。
   - 将 `图片展示` 文件夹中的所有文件和子文件夹拖拽到上传区域，或点击“选择你的文件”手动选择文件。
   - 上传完成后，点击“Commit changes”按钮完成上传。

## 步骤四：发布网站

1. 找到已经创建好的repository：
2. 依次点击add file → Upload files
3. 将该目录下的所有内容拖拽到指定区域（注意不要拖动整个根目录，而是打开根目录后的所有内容）

---

注意事项：

- 不要使用非ASCII字符（例如中文）。
